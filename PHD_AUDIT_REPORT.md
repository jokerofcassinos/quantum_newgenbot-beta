### BUG #5: Commission/Spread Calculation Double-Counts
**Severity:** 🔴 CRITICAL  
**Impact:** Costs overcharged by ~100x  
**Files:** All backtest runners

**The Bug:**
```python

costs = signal['volume'] * 45.0 * 2 + 100 * signal['volume']
```

**The Problem:**
- Commission: `volume * 45 * 2` = $0.90 for 0.01 lots ✓
- Spread: `100 * volume` = $1.00 for 0.01 lots ✗

For BTCUSD, spread should be `spread_in_dollars * volume`. If spread is $1:
- **Correct:** $1.00 × 0.01 = $0.01
- **Current:** 100 × 0.01 = $1.00 (100x too high)

**Fix:**
```python
spread_dollars = 1.0  # Typical BTCUSD spread
costs = volume * 45.0 * 2 + spread_dollars * volume
```
